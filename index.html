Conversational customer support agent on LangGraph — handles order queries, raises tickets, and escalates to a human agent, with per-user long-term memory and exact-checkpoint session resume.

  • Two-layer architecture — text-only core LangGraph StateGraph, with a swappable voice layer (STT + TTS) sitting on top
  • Six tools split into read-only lookups and write actions, with a code-level confirmation guardrail before any ticket creation or escalation
  • Per-user long-term memory persisted across sessions, with mid-session resume from the exact checkpoint
  • Sentence-streamed TTS output for natural-feeling voice responses
  • Text layer runs fully independently of voice — usable via plain text I/O, piped into an API, or pointed at any frontend, with the voice layer as an optional add-on
  • STT and TTS are each swapped by replacing a single function — no vendor logic baked into the core agent
  • Confirmation guardrail enforced at the code level inside the tools node — write actions are structurally blocked until the user gives an explicit affirmative, not just prompted for one
  • Long-term memory updated in place on session resume — each entry carries issue summary, resolution status, and behavioural notes for future sessions
  • Six tools split cleanly into read-only lookups (called freely) and write actions (ticket creation, escalation, memory edits) that all require confirmation
  • Ten-minute inactivity timeout auto-closes and saves the session, so state is never lost on an abandoned conversation
